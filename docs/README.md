# GenMURK — the TinyMUSE clean-room rebuild (c-build 2)

The studio's decompose-and-rebuild of a living system that can't run on modern
architecture. **Clean rebuild**, worked from decomposed requirements — the
original **TinyMUSE** engine (and the early-1990s MIT **MicroMUSE** instance) are a
**historical behavioral reference**, never adopted, never ported, never the
product's identity. Ruling of record: the STEERCO brownfield
decompose-and-rebuild ruling of 2026-07-10 (control plane).

- **`requirements.md`** — the decomposition (four subsystems) + requirements of
  record (GM-R1..R20) + rebuild backlog + architecture direction. The rebuild
  cites *these requirements*, never original source lines (provenance discipline).
- **`founding.md`** — founding record: naming/lineage, two-surfaces/two-domains
  (docs `genmurk.bussetech.com` / PROD `genmurk.com`), registry proposal,
  the `genmurk` knoll design, what EPIC5 executes, what is sysop-gated.
- **`gtm-artifact.md`** — the acceptance artifact: one GenMURK GTM piece produced
  from config alone by the generalized GTM gnomes (console-executed; live re-run
  filed).
- **`site/`** — the build-in-public living-docs site seed (instance 1 of
  `docs/build-in-public.md`), served at `genmurk.bussetech.com` once founded.
